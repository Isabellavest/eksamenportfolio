1) Teknik: HTML, CSS og mobile-first

I Tema 02 lærte jeg at bygge et website fra bunden. Jeg arbejdede med HTML som struktur og CSS som styling og layout — og jeg kodede mobile-first, så indhold prioriteres før dekoration.

HTML (struktur og semantik)

  • HTML er et markup-sprog der beskriver struktur og indhold
  • Semantiske tags: header, nav, main, section, footer
  • Fordele: bedre tilgængelighed, bedre SEO og lettere kodeforståelse

CSS (styling og layout)

  • CSS styrer layout, farver, fonte, spacing og responsivitet
  • Reset for ens udgangspunkt på tværs af browsere
  • Box model: content, padding, border, margin

Mobile-first

  • Jeg designer og koder først til mobil
  • Desktop kommer senere via breakpoints/media queries
  • Giver tydeligere struktur, bedre performance og stærkere UX

Dokumentation

2) Layout: Flexbox, Grid og media queries

Flexbox (1-dimensionelt)

Flexbox bruges til layout i én retning (række eller kolonne). Det er godt til navigation, cards i en række og simple layouts.

Grid (2-dimensionelt)

Grid bruges til layouts med både rækker og kolonner. I Tema 02 brugte jeg især Grid til at følge layoutdiagrammer og lave stabile desktop-layouts.

Responsivt design (media queries)

  • Jeg brugte breakpoints til at tilpasse layout fra mobil → desktop
  • Ved større skærme skifter layout typisk til grid/kolonner
  • Fokus på læsbarhed, luft og tydeligt hierarki

I layout-arbejdet lærte jeg også, at grid-classes kan genbruges (fx .grid_1-1) i stedet for at blive skrevet flere gange.

Dokumentation

3) Webfonte, billeder og performance

Webfonte

  • Man kan ikke regne med, at brugeren har samme fonte installeret
  • Google Fonts eller self-hostede fonte
  • WOFF2 er optimeret til web (TTF → WOFF2)

Billeder og filformater

  • JPG: fotos
  • PNG: transparens
  • SVG: vektor (skarpt i alle størrelser)
  • WEBP: god performance og kvalitet

Dokumentation

4) Design og analyse: gestalt, hierarki, typografi og farver

Udover kodning arbejdede jeg med at kunne analysere design og forklare valg med korrekt fagsprog. Det gjorde jeg gennem interface audits og stiløvelser (style tiles).

Gestaltlove

  • Nærhed: elementer tæt på hinanden opfattes som relaterede
  • Lighed: samme farve/form/stil skaber sammenhæng
  • Kontinuitet: øjet følger linjer og retninger
  • Lukkethed: hjernen udfylder manglende dele
  • Figur/baggrund: vi skelner mellem forgrund og baggrund

Visuelt hierarki

  • Størrelse, placering, kontrast og typografi guider brugerens blik
  • H1 → underoverskrift → brødtekst skaber naturlig læseretning

Farver og kontraster

Jeg arbejdede med farveharmoni og kontrast for overblik, usability og accessibility (fx komplementær, monokromatisk, mætning og varme/kulde).

Dokumentation

5) Studiestartsprøven: afleveringer og progression

Studiestartsprøven blev udviklet i flere trin. Mine afleveringer viser en tydelig progression: fra HTML-struktur → mobile-first styling → responsivt layout til desktop.

Afleveringer

  • Visitkortøvelse: Introduktion til Figma
  • Sæt indhold i HTML: 5 sider, semantik og korrekt indholdsrækkefølge uden CSS
  • Mobilsite: én CSS-fil + reset + farver + 2 fonte + spacing (mobile-first)
  • Website_v1 / Website: responsivt design, grid, media queries, to CSS-filer (styling/layout)
  • Stiløvelser: mini interface audit + style tile (gestalt/hierarki/typografi)

Samlet konklusion

Mine afleveringer viser en tydelig progression fra HTML-struktur til mobile-first CSS og videre til responsivt webdesign. Samtidig har jeg arbejdet analytisk med designprincipper, gestaltlove, typografi og visuel hierarki — så jeg både kan forklare teorien og vise den i praksis.
